USAGE NOTES
══════════════════════════════════════════════════════════════════════════════════════

1. Do NOT send the same letter to all three. Each one is calibrated to the specific
   technical problems that studio is known to be solving right now (April 2026).

2. Customize the [[ROLE_TITLE]] and [[TEAM]] brackets. Lazy copy-paste is the
   #1 reason cover letters get ignored.

3. If the job description mentions a specific technology you have in a project
   (FAISS, Claude, Diffusion, CLIP, etc.), swap the example project at the top of
   the letter to match. The letter should lead with the most relevant match.

4. Paste directly into the application form's "Cover Letter" field. Do not attach as
   a PDF unless the form explicitly requires one — applicant tracking systems parse
   plain text more reliably.

5. Always double-check the recipient name if you can find it on LinkedIn. "Dear
   [Netflix Hiring Team]" becomes "Dear [Hiring Manager Name]" if you can verify it.
